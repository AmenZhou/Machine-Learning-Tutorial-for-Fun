Feature Scaling Arena
💡 What is this game about?

You're teaching a computer to predict monthly AWS cloud bills. It uses two clues: how many EC2 servers you have, and how many database reads happened that month. The computer learns by adjusting two numbers (called "weights") until its guesses match the real bills. Your job: hit ▶ Auto and watch it learn.

The two inputs
🖥 EC2 servers: 1 to 10
🗄 DB reads: 10,000 to 200,000

⚠️ These are on completely different scales.
One is tiny (1–10). The other is massive (10k–200k).
That gap of 20,000× is the whole problem this week.

The analogy

Imagine a recipe that uses flour measured in tons and salt measured in pinches. You can't adjust both ingredients with the same scale — they need to be converted to the same unit first.

Feature scaling = converting everything to the same unit so the math works.

✅ What does "Standardized Mode" actually do?

It converts every input to a "how far from average, measured in standard deviations" score. The result is always a small number near zero — regardless of the original unit. The formula is:

z = ( value − average ) ÷ spread

❓ Why does Raw Mode fail?

The optimizer adjusts w₁ and w₂ by the same fractional step size. But the inputs they multiply are on completely different scales:

w₁ × servers    → w₁ × 3     (a small number)
w₂ × db_reads → w₂ × 120,000 (a huge number)

A tiny change in w₂ (e.g. 0.00001) causes a massive change in the prediction. A huge change in w₁ (e.g. +100) barely moves it. The loss landscape becomes a razor-thin canyon — nearly vertical on one side, nearly flat on the other. Gradient descent bounces off the walls and can't find the bottom.

💡 Switch to Standardized Mode to see how scaling fixes this.

Standardization converts both inputs to the same scale using z-score normalization:

z = (value − mean) ÷ std_deviation

servers: mean=5.0, std=3.0  → values become -1.3 to +1.7
db_reads: mean=77k, std=68k → values become -1.0 to +1.8

Now both inputs are in the same range (roughly -2 to +2). The loss landscape becomes a smooth round bowl — equal steepness in every direction. Gradient descent glides straight to the minimum in seconds.

✓ This is why feature scaling is always done before training in real ML projects. The model's math works the same — the inputs are just on a friendlier scale.
