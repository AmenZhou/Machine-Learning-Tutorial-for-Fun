📈 What is Linear Regression?

💡 In plain English

Linear regression is drawing the single best straight line through a cloud of data points — so you can predict where a new point will land.

The Problem It Solves

You have historical data: you know the server count and the actual monthly bill for 20 past months. Now a new month starts — you have 7 servers. What will the bill be?

Linear regression finds the line that best fits the past data, then uses it to answer that question.

# The model learns ONE formula:
bill = slope × servers + intercept

# Example after training:
bill = 280 × servers + 150

# Prediction for 7 servers:
bill = 280 × 7 + 150 = $2,110

The Two Parts of the Line

Slope

How much the prediction changes per unit of input. Slope = 280 means: every extra server adds £280 to the bill. A steep slope = strong relationship. Flat slope = weak relationship.

Intercept

The prediction when input = 0. The base cost even with zero servers — think of it as the fixed monthly fee (storage, networking, monitoring) that you pay regardless.

How It Connects to What You Already Know

From Phase 1: you learned that gradient descent slides a ball downhill to find the minimum loss. Linear regression uses exactly that — it adjusts slope and intercept step by step until the line fits as well as possible. The only difference: instead of one weight (x), you now have two (slope + intercept). Same algorithm, two numbers to tune.
The loss function: for each data point, measure the gap between the line's prediction and the real value. Square those gaps (so negatives don't cancel positives), then average them. That's Mean Squared Error (MSE) — the lower, the better the fit.

When to Use It

✅ Good for❌ Not suitable for
Predicting a number (cost, temperature, sales)Predicting yes/no (spam, churn, buy) — use logistic regression (Week 3)

What is MSE?

MSE = Mean Squared Error — the average of (prediction − actual)² across all data points. The red vertical lines on the chart are the individual errors. MSE is the average of all those gaps, squared. Squaring does two things: makes negatives positive, and punishes big errors more than small ones.

Lower MSE = better fit. MSE = 0 means the line passes through every point perfectly (never happens with real data). The goal is to get it as low as possible — that's what gradient descent does automatically.

What the Model is Actually Doing Each Step

// For each training step:
prediction = slope × x + intercept
error      = prediction − actual_bill
// Gradient descent update:
slope     = slope     − lr × mean(error × x)
intercept = intercept − lr × mean(error)

This is identical to the ball simulator from Phase 1 — just running for two weights (slope + intercept) simultaneously. Try the learning rate slider: set it high and watch it overshoot, low and watch it crawl.

📊 How Do You Know If the Model is Good?

💡 In plain English

MSE tells you the average error in squared units — hard to interpret. R² tells you the same thing as a percentage: "how much of the variation in bills does the model explain?"

R² Score — The Plain-English Score

1.0
Perfect — line passes through every point. Never happens with real data.
0.7
Good — model explains 70% of why bills go up and down. Acceptable for many real problems.
0.1
Poor — model barely explains anything. The feature may not predict the target at all.
Formula: R² = 1 − (MSE of your model) ÷ (MSE of just guessing the average every time)
In plain English: "how much better is the line than just guessing the average bill every month?"
