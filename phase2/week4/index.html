🛑 What is Regularisation?

💡 In plain English

Your model has 10 features. It uses all of them — including 5 noise features that have nothing to do with the bill. It memorises noise and fails on new data. Regularisation adds a penalty fine for complexity so the model stops cheating.

The Problem: Every Feature Gets a Coefficient

Without regularisation, the model assigns a coefficient to every feature — even the noise ones. It fits training data nearly perfectly (R² ≈ 0.99) by memorising the random noise patterns. On test data it has never seen, predictions collapse.

# What the model learns (bad — noise features get big weights):
bill = w1×servers + w2×db_reads + ... + w6×noise_1 + w7×noise_2 + ...

# Those noise weights are large — the model "memorised" them

The Fix: Add a Penalty for Large Coefficients

Think of it as a complexity tax. The model still tries to minimise MSE — but now it also pays a fine for large coefficients. Noise features can't justify their large weights because they don't reduce MSE enough to cover the fine. So they shrink.

# Without regularisation: just minimise MSE
Loss = MSE

# With regularisation: minimise MSE + penalty for big weights
Loss = MSE + α × (size of all coefficients)

# α controls how heavy the fine is — you dial it
Key insight: α = 0 → no penalty (same as standard linear regression). Large α → heavy fine → all coefficients driven toward zero. The trick is finding the sweet spot.

Two Types of Regularisation

L2 — Ridge Regression

Penalty = α × Σ w² (sum of squared coefficients).

Shrinks everything toward zero proportionally. Never forces a coefficient to exactly zero — it just makes them all smaller.

Best when: all features matter a little.

L1 — Lasso Regression

Penalty = α × Σ |w| (sum of absolute coefficients).

Forces irrelevant feature coefficients to exactly zero. Automatic feature selection — noise features simply disappear from the model.

Best when: you suspect many features are irrelevant.

Connection to Week 2: In Week 2, overfitting came from using a polynomial degree that was too high — the model bent its curve to fit every training point. Here, overfitting comes from too many features (10 features for 14 training examples). The root cause is the same: too much model complexity for the amount of data. Regularisation is the fix in both cases — it adds a constraint that prevents the model from going wild.

Real-World Example

A cloud cost model has 20 features. Engineers aren't sure which ones actually drive the bill. They apply Lasso — it automatically drops 15 irrelevant features and keeps only the 5 that matter. The model is now simpler, interpretable, and generalises better to new data.
